API Reference

This is the main documentation content for our Test Framework API.

Getting Started

To use the API, first install the package:

npm install test-api

Authentication

All API requests require authentication using an API key.

Set your API key: export API_KEY="your-key"

Note: Keep your API key secure and never commit it to version control!

Endpoints

The following endpoints are available:

  • GET /api/v1/users - List all users
  • POST /api/v1/users - Create a new user
  • GET /api/v1/users/{id} - Get user by ID
  • PUT /api/v1/users/{id} - Update user
  • DELETE /api/v1/users/{id} - Delete user

Response Format

All responses are returned in JSON format:

{
  "status": "success",
  "data": {
    "id": 123,
    "name": "John Doe"
  }
}

Status Endpoints

Method Endpoint Description
GET /api/status Check API status
GET /api/health Health check endpoint

Error Handling

The API uses standard HTTP status codes. Common errors include:

  • 400 Bad Request - Invalid request parameters
  • 401 Unauthorized - Missing or invalid API key
  • 404 Not Found - Resource not found
  • 500 Internal Server Error - Server error
